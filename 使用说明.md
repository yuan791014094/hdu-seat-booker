# 杭电图书馆自动抢座脚本 使用说明

## 文件清单

| 文件 | 用途 |
|------|------|
| `config.yaml` | 配置文件，填账号、座位、时间 |
| `book.py` | 主程序（登录 + 抢座） |
| `query_seats.py` | 查询所有座位ID |
| `查询座位.bat` | 双击运行 query_seats.py |
| `立即抢座.bat` | 双击立即抢座（不等定时） |
| `setup_task.bat` | 设置 Windows 每日定时任务 |

---

## 环境准备（只需做一次）

1. 安装 [Python 3.10+](https://www.python.org/downloads/)，安装时勾选 **Add Python to PATH**
2. 安装 [Google Chrome](https://www.google.com/chrome/)（已安装则跳过）
3. 打开命令提示符，运行：
   ```
   pip install selenium requests pyyaml
   ```

## 第一步：配置 config.yaml

用记事本打开 `config.yaml`，按需修改：

### 账号（无需改，已预填）

```yaml
account:
  username: "23060830"
  password: "Lqp20050125"
```

### 预约时间

```yaml
booking:
  date: "tomorrow"      # tomorrow=明天，today=今天
  begin_hour: 8         # 几点开始用座位（整点，24小时制）
  duration_hours: 4     # 连续使用几小时（1~8）
```

### 座位优先级列表

```yaml
seats_priority:
  - room_name: "格物E堂（二楼东）"
    seat_num: "424"
    seat_id: "62564"
    category_id: "607"

  - room_name: "格物E堂（二楼东）"
    seat_num: "422"
    seat_id: "62562"
    category_id: "607"
```

- 列表从上到下是优先级顺序，**第一个抢不到才抢第二个**
- `seat_id` 怎么查看下一步

### 定时抢座设置

```yaml
settings:
  trigger_time: "07:00"  # 每天几点自动触发抢座
  max_retries: 30         # 抢不到最多重试多少次
  retry_interval: 1       # 每次重试间隔秒数
```

> **trigger_time 怎么设？** 图书馆一般在某个固定时间点开放次日预约（如晚上8点或凌晨）。你可以先手动观察，把放票时间填进去，脚本会精确在那一刻开始抢。

## 第二步：查询座位ID（找你喜欢的座位）

**双击 `查询座位.bat`**，程序会自动登录并列出所有房间的座位状态，例如：

```
【格物E堂（二楼东）】category_id=607  可用94/424
  座位号     seat_id    状态
  001        62141      ✅可用
  002        62142      ❌占用
  003        62143      ✅可用
  ...
```

把你想要的座位的 `seat_num`、`seat_id`、`category_id` 填入 `config.yaml` 的 `seats_priority` 列表里。

## 第四步：设置每日自动定时抢座

**右键 `setup_task.bat` → 以管理员身份运行**

成功后会看到：
```
✅ 定时任务设置成功！
   任务名：杭电图书馆自动抢座
   每天 07:00 自动运行抢座脚本
```

之后每天到点电脑会自动运行脚本抢座，**电脑需要保持开机**。

> 可在开始菜单搜索"任务计划程序"查看或删除该任务。

## 常见问题

**Q：抢不到座位怎么办？**
- 增加 `max_retries` 重试次数
- 在 `seats_priority` 里多加几个备选座位
- 把 `trigger_time` 改得更准确（和放票时间一致）

**Q：登录失败？**
- 检查 `config.yaml` 里的账号密码是否正确
- 确认 Chrome 已安装

**Q：提示找不到 python / selenium？**
- 重新安装 Python 并勾选 Add to PATH
- 重新运行 `pip install selenium requests pyyaml`

**Q：如何修改定时触发时间？**
- 修改 `config.yaml` 里的 `trigger_time`
- 然后重新运行 `setup_task.bat`（管理员）
